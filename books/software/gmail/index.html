Gmail

2025-01-07

Gmail 中的 label 系统对我而言不好用

Gmail 的 label 系统是比较特殊的，相比其它邮件服务比如 Outlook、Yahoo Mail、Apple iCloud Mail 等都不相同。

在 Gmail 中所有邮件都存放在一个地方称为「所有邮件」，一封邮件可以有多个 label。收件箱也仅仅是一个 label，比如一封新来的邮件会被标记上「收件箱」label，当我们看完这封邮件，将其归档的时候其实是将这封邮件的「收件箱」label 移除而已，邮件本身不发生移动。而且你也只能新建标签而不是文件夹。其它邮件服务是将邮件存放在具体的文件夹中，一封邮件只能属于一个文件夹，归档操作是将邮件从具体文件夹中移动到归档文件夹。

那看起来 Gmail 更加灵活怎么会对我而言不好用呢？
我的使用场景是：我有多个收件箱，一般新来的邮件会被放入默认收件箱中，另一些邮件会被过滤到各自的收件箱中，处理邮件的时候通常会即时处理默认收件箱中的，处理完进行归档（放入归档文件夹中），具体收件箱可以根据不同紧急程度等进行创建，一些不紧急的邮件可以过一段时间进行统一处理（也是处理完放入归档文件夹中），但是又不至于污染默认收件箱。

这在其它邮件服务上很容易实现：创建一些文件夹用于收取邮件，对邮件进行过滤将其放入不同的收件箱中，没有被过滤掉的邮件放入默认收件箱中，任何被处理好的邮件点击归档按钮会被自动移动到归档文件夹中。

但在 Gmail 上面却无法实现，比如我创建一些标签用于收取邮件，然后创建一些过滤给不同的邮件打上不同的标签，当我在默认收件箱中归档是没有问题的，会被移除「默认收件箱」label，那么我在收件箱中就看不见这封邮件了。但是其它收件箱却实现不了归档功能，因为归档功能是移除「默认收件箱」label，但是其它收件箱是个标签，为了让邮件出现在其它收件箱而不是默认收件箱不会被打上「默认收件箱」label，也就不支持移除「默认收件箱」label 这个功能，也就是归档功能在其它收件箱中无法实现。

但是有时候不得不使用 Gmail 邮箱来接收邮件的时候一个简单的解决办法是：在 Gmail 设置中将需要的邮件转发到其它邮件服务中 :)
