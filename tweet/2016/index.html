govendor 这个包是什么鬼。

Some examples:
- github.com/kardianos/govendor specifies a single package and single folder.
- github.com/kardianos/govendor/^ specifies the govendor folder and all sub-folders. Useful for resources or if you don’t want a partial repository.

默认竟然是 fetch 单个包和单个文件夹，如果某个包有子文件夹就获取不到了，太扯了吧。 按照正常的思维谁会想到是这个问题啊，害我在部署 Heroku 的时候浪费了很多时间， 原因竟然是文件不全。。。

使用 Vim 能够实现对字符的完全控制，用起来感觉是不错。

英文中登录登出有两组词可用
sign in, sign out 和 log in, log out
对各大网站的统计来看，sign in, sign out 更常用

注册有 register, sign up 可用
注册登录用 register, sign in 更好，因为这两个单词相差很大，用户好辨识

结论：
登录，登出，注册使用 sign in, sign out, register

用了那么久的日历 （iOS 系统自带）功能，到今天我才发现我用错了

比如我今天下午下班回来的时候要去超市买支牙膏，于是我设定下午六点提醒我，六点的时候日历确实提醒我了，但是这一天还是忘了买了。结果这个事件虽然还在日历中存在，但是不会再进行任何提示，最后的结果就是忘记买牙膏，直到无牙膏可挤为止 这种需求应该用提醒事项来做，就算事情过期了还是会提醒你，直到你完成为止，不至于忘记做这件事

总结一下，日历一般用于计划好的且对时间要求比较严格，像是上课日程表之类的
提醒事项用于对时间要求不那么严格的，注重在提醒
